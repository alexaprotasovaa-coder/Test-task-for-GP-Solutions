## Task Description

This project was completed as a **system analysis exercise**.

The assignment required analyzing a video that demonstrated the functionality of a travel distribution platform and producing structured documentation describing the system.

The task included:
- reviewing the system workflow presented in the video
- identifying key business entities and user roles
- describing the distribution model of travel products
- documenting pricing visibility rules
- describing the booking lifecycle and voucher generation

The final deliverable had to be written in English and prepared for **two different audiences**.

---

# Deliverables

Two separate documents were created as a result of the assignment.

### 1. System Overview for External Users

Audience:
- Tour Operators
- Travel Agencies
- Other system partners

Purpose:
- explain how the platform works from a **business user perspective**
- describe the distribution model and user roles
- explain how bookings and vouchers work
- clarify pricing visibility rules between partners

This document focuses on **system usage and business logic**, without internal implementation details.

---

### 2. System Overview for Internal Users

Audience:
- Business Analysts
- Developers
- QA Engineers

Purpose:
- provide a **high-level understanding of the system architecture**
- document key concepts used in the platform
- describe user roles and the multi-level distribution mode
- define pricing visibility rules across the sales chain
- outline the booking lifecycle and voucher logic

This document is designed to support **team onboarding and understanding of core system logic**.

---

# Key Topics Covered

The documentation describes the following aspects of the system:
- multi-level **distribution chain** of travel products
- **roles participating in the system** (supplier, tour operator, agency, customer)
- **pricing visibility rules** between participants
- **booking lifecycle**
- **voucher generation and content rules**
- high-level **system modules**

---

# Result

As a result of this assignment, a structured system overview was produced that:
- clearly defines the **distribution model of the platform**
- documents **roles and relationships between participants**
- explains **pricing transparency rules**
- describes the **booking and voucher flow**

The documentation is intended to help both **business users and technical team members** quickly understand how the platform operates.
